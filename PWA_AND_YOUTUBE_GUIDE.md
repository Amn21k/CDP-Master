# CDP Master by Dr. Aman - PWA & Backend Integration Guide

## 1. PWA (Progressive Web Application) Checklist
The application has been successfully configured as a PWA using `vite-plugin-pwa`. It includes offline caching mechanisms with `workbox` and an `autoUpdate` service worker.

### How it works:
- **Offline Caching:** The plugin automatically caches all generated static assets (`js`, `css`, `html`). Your Google Fonts are also automatically cached.
- **Manifest:** A fully configured manifest allows the web app to be "installed" on mobile homescreens with native app-like experience. Fast initial loads are supported via local cache fallback.

### Next Steps for Deployment:
1. Ensure your `public` folder contains the necessary icons: `favicon.ico`, `android-chrome-192x192.png`, and `android-chrome-512x512.png`.
2. Deploy the `dist` folder generated by `npm run build` to a static hosting provider like Firebase Hosting, Vercel, or Netlify.
3. Access from Android (Chrome) or iOS (Safari) and tap **"Add to Home Screen"**.

## 2. Question Bank Database (Schema Guide)

Currently, the application uses local JSON/TypeScript mock data in the `src/data.tsx` file for instant testing. However, for a production-level 'CDP Master' question bank, migrate to a Cloud Database (e.g., Firebase Firestore or PostgreSQL).

### Recommended Question Schema (JSON/CSV Format)
For seamless integration into the app, maintain your question bank in the following structure:

```json
{
  "questions": [
    {
      "id": "q_101",
      "topicId": "piaget_cognitive",
      "questionText": "जीन पियाजे के अनुसार, एक बच्चा जो यह मानता है कि बादलों में भी जान होती है, वह किस विशेषता को दर्शाता है?",

        "अमूर्त चिंतन"
      ],
      "correctAnswerIndex": 1,
      "difficulty": "medium",
      "explanation": "जीववाद (Animism) पूर्व-संक्रियात्मक अवस्था (2-7 वर्ष) की विशेषता है, जहाँ बच्चा निर्जीव वस्तुओं को सजीव मानता है।",
```

**Importing to App:**
Use a serverless backend to query a dynamic 30 randomly selected question batch by mapping over the question documents.

## 3. YouTube Data API v3 conceptual guide for "Magic Tricks"
You explicitly requested an architecture to automate or analyze YouTube pedagogy videos to generate the mnemonics used in the "💡 जादुई ट्रिक्स (Magic Tricks)" section.

Here is the conceptual flow for building this backend service:

### Step A: Setup YouTube Data API
1. Navigate to the **Google Cloud Console**.
2. Enable the **YouTube Data API v3**.
3. Generate an API Key.

### Step B: Fetch Captions / Transcripts
Rather than analyzing video pixels, your backend should query leading YouTube educators' channels for videos containing "CDP Tricks", "UPTET Tricks" etc.
   - Use the `Search: list` endpoint: `GET https://youtube.googleapis.com/youtube/v3/search?part=snippet&q=CDP+Piaget+Tricks+UPTET`
   - Extract the generated `videoId`.
   - Access the video's subtitle/transcript payload using a library like `youtube-transcript-api` (Python/Node).

### Step C: Expert AI Analysis via Google Gemini
Pass the raw Hindi transcript text to the Google Gemini API with a targeted system prompt:
```text
System Prompt: You are a CDP pedagogical expert. 
Read the following transcript text from a UPTET educator.
Extract only the 'memory tricks', mnemonics, or abbreviations taught in the video.
Output them strictly in the Magic_Trick_Schema JSON format.
```

### Step D: Frontend Integration
Store the Gemini-processed Flashcard schema in a database.
The frontend `Flashcard UI swipe-layout` currently maps `FLASHCARDS` array from local state. Once the backend is built, replace `FLASHCARDS` with an API call `fetch('/api/magic-tricks')` and set the response to state, seamlessly displaying the most up-to-date YouTube-analyzed shortcuts.
